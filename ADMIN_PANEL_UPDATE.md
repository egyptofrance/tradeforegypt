## ملاحظات تقنية:

### لماذا Inline CSS؟
- Tailwind CSS يتطلب build process
- قد تكون هناك مشاكل في التكوين
- Inline CSS يضمن العمل الفوري
- سهل الصيانة والتعديل

### البيانات المطلوبة في Supabase:
- جدول `brands` مع الأعمدة: logo_url, banner_url, cover_url, description, meta_title, meta_description
- جدول `products` مع الأعمدة: photo_1, photo_2, photo_3, photo_4, photo_5, description, meta_title, meta_description
- جدول `families` موجود
- جدول `brand_families` للعلاقات (many-to-many)

### Supabase Storage Buckets المطلوبة:
- `brands` (public)
- `products` (public)
- `defaults` (public) - للصور الافتراضية
